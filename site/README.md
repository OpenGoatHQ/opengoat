# opengoat-site

The opengoat.com Next.js site. Reads humans and playbooks from `../data/index.json` (auto-generated by `scripts/build-index.ts` on every push to main).

## Stack

- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS + `@tailwindcss/typography`
- `react-markdown` + `remark-gfm` for playbook bodies
- `fuse.js` for client-side search

## Pages

| Route | What it renders |
|---|---|
| `/` | manifesto, categories, recent humans, recent playbooks |
| `/manifesto` | full thesis |
| `/cli` | CLI quick reference |
| `/mcp` | MCP server setup for AI agents |
| `/contribute` | how to be listed + bar |
| `/search` | client-side fuzzy search |
| `/c/<category>` | category landing + operators grid |
| `/<handle>` | operator profile + playbook list + booking CTA |
| `/playbooks/<slug>` | full playbook with skill manifest cards |

## Local dev

```bash
cd site
npm install
npm run dev
```

Site runs at <http://localhost:3000>. Data is read from the parent repo's `data/index.json`.

## Deploy

Vercel project pointing at this repo root. The `vercel.json` at the repo root tells Vercel to build `site/`. Auto-deploys on push to `main`.

Set the production domain to `opengoat.com` in Vercel project settings.

## Refresh data

Whenever a PR adding a human or playbook is merged, the `build-index` GitHub Action rebuilds `data/index.json` and commits it back. Vercel sees the commit and redeploys.
