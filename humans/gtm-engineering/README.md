# GTM Engineering

Operators who build the modern GTM stack: Clay workflows, reverse ETL (Census, Hightouch), CDPs (Segment, RudderStack), automation (n8n, Make, Zapier), attribution (HockeyStack, Dreamdata), and AI agents in revenue workflows.

## What we look for

- Real implementations they've shipped (not just configured demos)
- Comfort with both code and no-code tools
- Understanding of where GTM eng fails (usually data hygiene, not tooling)

## Anti-fits

- "GTM engineers" who only know one tool
- Anyone who can't articulate trade-offs between Clay/Apollo/etc.
